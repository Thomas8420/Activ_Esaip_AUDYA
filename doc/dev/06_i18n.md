# 06 — Internationalisation (i18n)

## Architecture

| Fichier | Rôle |
|---------|------|
| `src/i18n/translations.ts` | Toutes les chaînes FR / EN / ES |
| `src/context/LanguageContext.tsx` | Context React + hook `useLanguage()` |

---

## Langues supportées

| Code | Langue | Complète |
|------|--------|----------|
| `FR` | Français | ✅ (langue par défaut) |
| `EN` | English | ✅ |
| `ES` | Español | ✅ |

---

## Utiliser `t()` dans un composant

```tsx
import { useLanguage } from '../../context/LanguageContext';

const MyComponent = () => {
  const { t } = useLanguage();

  return <Text>{t('settings.save')}</Text>;
};
```

Le fallback est automatique : si une clé n'existe pas dans la langue courante, `t()` retourne la valeur FR, puis la clé brute si introuvable.

---

## Changer la langue

```tsx
const { setLanguage } = useLanguage();

// Changement immédiat — l'app entière se re-render
setLanguage('EN');
```

`setLanguage` met à jour une variable module-level (`_persistedLanguage`) avant de setter le state React. La langue survit aux démontages de composants mais pas aux redémarrages de l'app (TODO: AsyncStorage).

## Ajouter une clé de traduction

1. Ouvrir `src/i18n/translations.ts`
2. Ajouter la clé dans les **3 langues** (FR, EN, ES) dans la section appropriée :

```typescript
// Dans FR:
'mon.nouveau.label': 'Mon label',

// Dans EN:
'mon.nouveau.label': 'My label',

// Dans ES:
'mon.nouveau.label': 'Mi etiqueta',
```

3. Utiliser dans le composant : `t('mon.nouveau.label')`

---

## Ajouter une langue

1. Dans `src/i18n/translations.ts` :

```typescript
// Ajouter au type
export type Language = 'FR' | 'EN' | 'ES' | 'DE';

// Créer le bloc de traduction
const DE: TranslationMap = {
  'menu.health': 'Meine Gesundheit',
  // ... toutes les clés
};

// L'exporter
export const TRANSLATIONS: Record<Language, TranslationMap> = { FR, EN, ES, DE };
```

2. Dans `src/context/LanguageContext.tsx` : aucun changement nécessaire (le type est importé).

3. Dans `src/components/Settings/SettingsPage.tsx` :

```typescript
const LANGUAGE_OPTIONS: Language[] = ['FR', 'EN', 'ES', 'DE'];
```

## Interpolation de variables

La fonction `t()` ne supporte pas l'interpolation native. Pour les strings avec variables (ex: copyright avec l'année) :

```tsx
// Pattern actuel : remplacement manuel post-traduction
t('main.footer.copyright').replace('{year}', String(new Date().getFullYear()))
```

La clé contient le placeholder `{year}` dans toutes les langues :
```typescript
'main.footer.copyright': '© {year} AUDYA — Tous droits réservés',
```
